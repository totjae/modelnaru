# ModelNaru Web UI 명세

## 1. 목적

ModelNaru의 로그인, 사용자 작업공간, 채팅과 관리자 화면이 같은 시각 언어를 사용하도록 공통 UI 기준을 정의한다.

## 2. 적용 범위

Next.js Web의 모든 화면, 공통 컴포넌트, 반응형 레이아웃과 상태 표현에 적용한다.

## 3. 상세 명세

### 3.1 기본 콘셉트

- 다크 테마는 검정, 라이트 테마는 흰색과 무채색을 기반으로 한다.
- 넓은 면적에는 녹색이나 7색 혼합 gradient 배경을 사용하지 않는다.
- 정보 밀도가 높은 도구형 UI를 지향하되 충분한 여백과 명확한 계층을 유지한다.
- 7색은 각 기능과 상태에 하나씩 배정하는 브랜드 포인트다. 한 요소에는 원칙적으로 한 가지 포인트 색만 사용한다.
- 7색을 섞은 무지개 gradient는 사용하지 않는다.
- 본문과 넓은 카드 배경에는 유채색을 채우지 않는다.

### 3.2 색상 토큰

| 용도        | 라이트                                   | 다크                        |
| ----------- | ---------------------------------------- | --------------------------- |
| 전체 배경   | `#ffffff`                                | `#050505`                   |
| 기본 표면   | `rgba(255, 255, 255, 0.96)`              | `rgba(14, 14, 15, 0.94)`    |
| 강한 표면   | `#f7f7f8`                                | `#101011`                   |
| 기본 테두리 | `rgba(0, 0, 0, 0.14)`                    | `rgba(255, 255, 255, 0.13)` |
| 기본 글자   | `#111114`                                | `#f5f5f7`                   |
| 보조 글자   | `#686870`                                | `#9a9aa2`                   |
| 기본 강조   | `#111114`                                | `#f4f4f5`                   |
| 오류        | `#c93636`                                | `#ff8f8f`                   |
| 7색 순서    | 빨강, 주황, 노랑, 초록, 청록, 파랑, 보라 | 라이트 테마와 동일          |

7색은 각각 독립된 공통 token으로 관리한다.

| 색상 | 기본 역할                        |
| ---- | -------------------------------- |
| 빨강 | 삭제, 실패, 위험                 |
| 주황 | 관리자, 주의                     |
| 노랑 | 게스트                           |
| 초록 | 성공, 정상 동작                  |
| 청록 | 일반 사용자·사용자 관리          |
| 파랑 | 기본 동작, 접근 설정, 공통 focus |
| 보라 | 브랜드, Provider, 테마 설정      |

새 화면도 의미가 같은 기존 색을 재사용한다. 장식 목적으로 여러 색을 한 요소에 겹치거나 화면마다 역할을 바꾸지 않는다.

브랜드 아이콘은 보라 `#7c3aed`를 단일 주색으로 사용한다. 중립적인 밝은색 심벌만 조합하고 gradient·그림자·추가 장식은 넣지 않는다. 둥근 사각형 바깥은 투명하게 두어 밝은 배경과 어두운 배경에서 같은 파일을 사용한다. 심벌은 일반 글꼴이 아닌 기하학적 `MN` 모노그램으로 만들고 가운데 세로 기둥 없이 한 줄로 연결한다.

원본은 `apps/web/public/modelnaru-icon.svg`로 관리한다. Web 화면의 브랜드 마크, favicon, Apple touch icon과 PWA manifest에 같은 심벌을 사용하며 PNG·ICO 파생 파일은 `tools/generate-brand-icons.py`로 다시 생성할 수 있어야 한다. 청록과 주황은 각각 사용자·관리자 UI 역할색으로 유지하고 대표 아이콘 배경에는 사용하지 않는다.

브라우저·운영체제 아이콘은 작은 크기 식별성을 위해 보라색 면을 채운다. 페이지 내부 브랜드 마크는 같은 MN 심벌을 사용하되 `1.75px` 보라색 테두리, 반투명 중성 표면과 제한적인 blur로 글라스 형태를 만들고 내부 심벌만 테마별 `--violet` 색상을 사용한다.

### 3.3 테마 동작

- 선택지는 `시스템`, `라이트`, `다크` 세 가지다.
- 최초 접속은 운영체제의 `prefers-color-scheme`을 따른다.
- 사용자가 고른 모드는 브라우저 local storage에 저장하고 다음 방문에도 적용한다.
- `시스템` 선택 중에는 운영체제 테마가 바뀌면 화면도 즉시 전환한다.
- 첫 화면 표시 전에 저장된 테마를 적용해 밝은 화면이 순간적으로 번쩍이는 현상을 방지한다.

### 3.4 컴포넌트

- 기본 버튼은 밝은 무채색 배경과 검정 글자를 사용한다.
- 보조 버튼은 중성 표면과 테두리를 사용하며 hover 때 해당 기능에 배정된 단색 테두리를 표시할 수 있다.
- 입력창은 중성 표면과 테두리를 사용하고 focus 상태에서 해당 기능의 단색 테두리를 표시한다.
- 카드의 기본 테두리는 중성색이며 로그인 카드, 현재 선택된 항목 등 의미 있는 곳에만 역할별 단색 테두리를 사용한다.
- 성공 상태는 흰색과 중성 표면을 기본으로 하고, 오류·위험 상태에는 제한적으로 빨강을 사용한다.
- 체크박스와 radio는 고정 크기로 정렬하며 주변 입력창의 전체 너비 규칙을 상속하지 않는다.

### 3.5 레이아웃

- 데스크톱은 관리·설정 영역을 2열까지 사용하고 860px 이하에서는 1열로 전환한다.
- 관리자 로그인 직후 기본 메뉴는 `Usage`다. 상단 메뉴는 `Usage → 사용자 → 게스트 → 프로바이더 → 장기기억 → 서버` 순서이며 한 번에 선택한 메뉴의 내용만 표시한다.
- 사용자 메뉴에는 계정 생성·수정·삭제와 사용자별 모델 권한을 함께 두고, 게스트 메뉴에는 공유 코드·세션·호출 제한·모델 권한을 둔다. 프로바이더와 장기기억 메뉴는 각각 Provider 연결과 컨텍스트 자동 요약 설정을 표시한다.
- Provider 모델 목록의 활성화 버튼은 `사용 중` 문구와 초록색 테두리·옅은 초록 표면을 함께 사용해 비활성 모델과 구분한다. `사용 안 함` 버튼은 중립색을 유지한다.
- 서버 메뉴의 첨부파일 보관 화면은 보관 일수 설정과 자동 정리 상태를 2열 카드로 표시하고 860px 이하에서 1열로 전환한다. 보관 파일 수·크기, 삭제 대기, 최근 실행·결과와 `지금 정리` 동작을 제공한다.
- 만료된 메시지 attachment는 기존 파일명·크기·페이지·해상도를 유지하면서 `원본 만료` 문구를 추가한다.
- Usage는 `10분 · 1시간 · 6시간 · 12시간 · 1일 · 1주 · 30일` 상대 기간을 제공하고 기본값은 1일이다. 전체 token·요청·성공률·사용 주체 요약과 사용자별·모델별 표, 대화·자동 요약 유형을 구분한 최근 요청 50건을 표시한다.
- Usage 표는 좁은 화면에서 자체 가로 스크롤하며 페이지 전체 너비를 밀어내지 않는다. 기능별 요약 카드는 860px 이하 2열, 520px 이하 1열로 전환한다.
- 채팅 화면은 대화 목록과 대화 본문의 2열을 기본으로 하며 모델·파라미터 설정은 독립 모달로 연다.
- 장시간 대화 사용을 고려해 본문 영역에는 강한 gradient나 지속적인 광원 효과를 사용하지 않는다.
- 데스크톱 채팅은 대화 목록·메시지 본문의 2열이며, 520px 이하에서는 전체를 1열로 전환한다.
- 사용자 메시지는 청록, AI 메시지와 생성 parameter는 보라, 새 대화는 파랑, 전송은 청록, 설정 저장은 초록, 중지·삭제는 빨강의 단색 역할을 사용한다. 주요 동작 버튼은 넓은 유채색 면 대신 해당 역할색의 테두리와 글자를 사용한다.
- 왼쪽 대화 목록은 접고 펼칠 수 있으며, 접으면 `새 대화` 버튼과 목록·격리 안내를 포함한 사이드바 전체를 숨기고 대화 본문이 전체 너비를 사용한다. 설정은 기본 닫힘 상태이며 상단 `설정` 버튼으로 모달을 연다.
- 왼쪽 대화 목록의 각 대화방 행은 항목 수와 관계없이 68px 고정 높이를 사용한다. 남는 세로 공간을 행에 분배하지 않으며 목록이 영역을 넘을 때만 내부 스크롤한다.
- 각 대화방 행에는 독립된 `삭제` 동작을 제공한다. 확인 후 대화·분기·메시지를 삭제하며 현재 대화를 삭제하면 남은 첫 대화로 이동하고, 다른 대화를 삭제하면 현재 대화를 유지한다.
- 채팅 상단 패널 조작 바는 메시지 영역을 과도하게 차지하지 않도록 약 46px의 조밀한 높이를 사용한다.
- 브라우저 높이가 낮거나 대화방 수가 많아져도 입력창과 전송·중지 버튼의 높이는 축소하지 않는다. 채팅 Grid 행은 동적 viewport(`dvh`) 안에서 늘어나지 않게 제한하고, 왼쪽 대화 목록과 가운데 메시지 목록만 각 영역 안에서 독립적으로 스크롤한다. 사이드바 내용 높이가 대화 본문과 입력창 위치를 밀어내서는 안 된다.
- 설정 모달은 데스크톱에서 최대 너비 860px·최대 높이 84vh를 사용하고 본문만 세로 스크롤한다. 입력 순서는 `대화 제목 → 모델 → 생성 파라미터 → 이전 메시지 수 → 컨텍스트 토큰 한도 → 시스템 프롬프트`로 고정한다.
- 설정 모달 하단에는 `취소 → 대화 삭제 → 설정 저장` 동작을 고정한다. 바깥 영역 클릭과 `Esc`로 닫을 수 있으며 저장하지 않은 변경이 있으면 확인한 뒤 모델·파라미터를 열기 전 상태로 복원한다.
- 520px 이하에서는 설정 모달을 화면 전체 크기로 표시하고 저장 버튼을 가장 먼저 인식할 수 있도록 전체 너비로 배치한다.
- 생성 파라미터는 이름, 쉬운 설명, `기본값: Provider 자동 설정` 안내, `직접 설정` 체크박스, 설정 입력 순서로 표시한다. 직접 설정하지 않은 입력도 사라지지 않고 비활성 상태로 유지한다.
- reasoning·thinking처럼 다른 설정과 함께 적용할 수 없는 파라미터도 숨기지 않는다. 회색 비활성 상태와 적용되지 않는 이유를 해당 항목 안에 표시한다.
- 이전 메시지 수에는 `0은 전체 대화`라는 뜻과 값이 클수록 더 많은 이전 문맥을 전달한다는 짧은 설명을 표시한다.
- 이전 메시지 수 설명은 `0은 전체 대화입니다.` 다음에 줄을 바꿔 추가 설명을 표시한다.
- 사용자·게스트 채팅 상단은 아이콘, `ModelNaru Workspace`, 공간 이름, 계정 ID와 로그아웃을 한 줄에 배치한다. 아이콘은 36px, 헤더 아래 여백은 약 12px의 조밀한 높이를 기본으로 하며 좁은 화면에서는 글자를 줄이고 공간 이름·ID를 말줄임한다.
- 기존 대화를 선택하거나 새로고침하면 해당 대화에 저장된 기본 모델과 생성 파라미터를 설정 모달에 복원한다. 저장 모델이 현재 권한에서 제외되면 활성 분기의 마지막 허용 모델, 그 다음 첫 허용 모델 순서로 대체한다.
- 설정 저장은 현재 대화 UUID에만 적용한다. 제목이 같은 다른 대화와 모델·생성 파라미터 상태를 공유하지 않는다.
- 현재 활성 경로의 가장 마지막 AI 답변에만 아이콘 형태의 재생성 버튼을 제공한다. 재생성 중에는 새 답변을 기존 위치에서 미리 보여주고 성공 후 별도 분기로 전환한다.
- 상단 전역 분기 선택기는 사용하지 않는다. 마지막 AI 답변 아래에 `이전 답변 · 현재/전체 · 다음 답변 · 재생성` 순서의 인라인 조작을 표시한다.
- 좌우 답변 탐색은 같은 사용자 메시지를 부모로 갖는 정상 선택 가능 답변만 묶는다. 실패·취소된 재생성 분기와 다른 질문에서 갈라진 분기는 후보에서 제외한다.
- 메시지를 보내거나 대화·답변 분기를 전환하면 최신 메시지가 보이도록 본문을 아래로 이동한다. 스트리밍 중 목록이 하단 96px 이내면 새 응답을 계속 따라가고, 사용자가 그보다 위로 스크롤하면 자동 추적을 중단한다.
- 응답 완료와 재생성 완료 후에는 메시지 상세와 대화 목록만 백그라운드에서 갱신한다. 초기 진입용 전체 로딩 화면으로 전환하거나 채팅 작업공간을 교체하지 않는다.
- 설정 저장·삭제·분기 전환·오류 같은 상태 메시지는 본문 배치를 밀어내는 고정 영역 대신 채팅 상단의 떠 있는 토스트로 표시한다. 토스트는 약 5초 동안 유지하며 등장·종료 시 짧게 fade하고 자동으로 사라진다.
- 입력창 위 `파일 추가`에서 지원 텍스트 파일, PDF와 JPEG·PNG·WebP 이미지를 최대 10개까지 올릴 수 있다. 전송 전 목록은 파일명·크기·PDF 페이지 수·이미지 해상도·개별 `후속 메시지에도 포함` 체크와 삭제를 제공하며, 전송된 user 메시지에는 원본을 노출하지 않는 attachment chip을 표시한다.
- PDF 페이지 초과·암호 보호·OCR 필요·손상 상태는 사용자가 다음 행동을 알 수 있는 한국어 토스트로 구분한다.
- 현재 선택 모델의 `이미지 입력` capability가 꺼져 있으면 이미지가 있는 메시지를 보내지 않고 이미지 지원 모델을 선택하라는 토스트를 표시한다.
- 사용할 수 없는 답변 탐색 화살표는 회색과 낮은 불투명도를 유지하되 금지 모양 cursor를 사용하지 않는다.

## 4. 오류·예외와 경계 조건

- 포인트 색상만으로 오류·성공·비활성 상태의 의미를 전달해서는 안 된다.
- 텍스트 대비는 WCAG AA를 목표로 하며 색상만으로 상태를 전달하지 않는다.
- 포인트 색상에 지속적인 점멸이나 색상 순환 애니메이션을 사용하지 않는다.
- 비활성 요소는 흐리게 표시하되 글자를 읽을 수 있는 대비를 유지한다.

## 5. 검증·인수 조건

- 다크에서는 검정, 라이트에서는 흰색 기반 표면이 사용되고 두 모드 모두 역할별 단색 포인트를 유지한다.
- 시스템 테마 추종, 수동 전환과 선택값 복원이 동작한다.
- 포인트 색상이 기능별 7색 역할표와 일치하고 무지개 gradient가 없다.
- 키보드 focus가 명확하며 체크박스 정렬이 입력창 너비 규칙에 영향받지 않는다.
- 860px와 520px 반응형 구간에서 가로 스크롤 없이 주요 기능을 사용할 수 있다.
- 대화 목록을 접고 다시 펼칠 수 있으며, 설정 모달은 기본 닫힘·독립 스크롤·바깥 클릭·Esc·미저장 확인 동작을 제공한다.
- 대화방 수가 적어도 목록 행 높이가 늘어나지 않고, 목록의 삭제 버튼으로 현재 또는 다른 대화를 안전하게 삭제할 수 있다.
- 대화마다 서로 다른 모델과 생성 파라미터를 저장한 뒤 새로고침하거나 대화를 왕복해도 각 대화 설정이 독립적으로 복원된다.
- 답변을 재생성해도 원본을 다시 선택할 수 있고 분기 전환 후 새로고침해도 활성 분기가 유지된다.
- 과거 AI 답변에는 재생성·분기 아이콘이 없고 마지막 AI 답변에서만 `1/1`, `1/2` 형식의 탐색 상태가 표시된다.
- 메시지 전송 후 첫 응답과 스트리밍 중인 최신 내용이 화면에 유지되며, 과거 내용을 읽으려고 위로 스크롤하면 화면을 강제로 하단으로 이동시키지 않는다.
- 응답 또는 재생성이 완료될 때 작업공간 전체가 로딩 화면으로 바뀌거나 새로고침처럼 깜빡이지 않는다.
- 상태 토스트가 채팅 도구 모음과 본문 위치를 바꾸지 않고 약 5초 후 자동으로 사라진다.
- 텍스트 attachment를 전송 전 추가·설정·삭제할 수 있고 파일만으로도 메시지를 보낼 수 있으며 새로고침 뒤 user 메시지에 metadata가 유지된다.
- production build와 정적 검사가 통과한다.

## 12. Provider 등록 필드와 채팅 기본 Temperature

- Provider 선택에 따라 API 키 필수 여부와 template별 추가 필드를 동적으로 표시한다.
- `bearer-optional`과 `none` 인증은 API 키가 선택 사항임을 표시한다.
- Cloudflare AI Gateway 선택 시 Account ID 입력란을 표시한다.
- 새 채팅 workspace의 Temperature 직접 설정은 기본으로 활성화하며 값은 `1.0`이다.
- 모델이 Temperature를 지원하지 않거나 다른 설정과 충돌하면 요청에서는 제외하고 비활성화 이유를 표시한다.
